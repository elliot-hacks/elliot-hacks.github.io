HackTheBox: OpenAdmin

Linux Easy OpenNetAdmin SSH Nano Privesc

Step 1: Service Enumeration

Initial Nmap scan reveals SSH and HTTP services:

[msf] >> db_nmap -p- -T5 --min-rate=5000 -Pn openadmin.htb
PORT   STATE SERVICE
22/tcp open  ssh
80/tcp open  http

Version scan reveals Apache2 default page:

80/tcp open  http    Apache httpd 2.4.29 (Ubuntu)
22/tcp open  ssh     OpenSSH 7.6p1 Ubuntu

Step 2: Directory Bruteforce

Using ffuf to discover directories:

┌─[elliot@parrot]─[~]

Step 8: Cracking SSH Private Key

┌─[elliot@parrot]─[~]
└──╼ $john --format=SSH --wordlist=/usr/share/wordlists/rockyou.txt openadmin_hash

Using the cracked passphrase to connect as Joanna:

┌─[elliot@parrot]─[~]
└──╼ $chmod 600 openadmin_hash

└──╼ $ssh -i openadmin_hash joanna@openadmin.htb

Key Takeaways

  • OpenNetAdmin RCE - 18.1.1 vulnerable to command injection
  • Credential Reuse - Database password reused for SSH user
  • Internal Port Scanning - Discovering hidden services
  • SSH Key Cracking - Using John the Ripper for private key passphrases
  • Nano GTFObins - sudo nano privilege escalation technique

This machine demonstrates a complete attack chain from web application RCE to domain privilege escalation.
